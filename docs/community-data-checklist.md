# Community data checklist

Tracks the standard data points every villa/townhouse community should carry. Fill the ⬜ items manually or via the **Absorb** tab (upload brochure/price list → extract → approve → live). Auto-updated fields: **Pin** and **Narrative** are done for all 100 communities.

**Standard data points per community**
1. **Pin** — map coordinate (done for all)
2. **Narrative** — description + who-it's-for (done for all; refine per community)
3. **Clusters** — sub-communities present
4. **Counts** — villa / townhouse / total units
5. **Units & sizes** — unit archetypes with BUA / plot / beds / baths
6. **Prices** — guide/from price (indicative) and, once DLD is connected, sold transactions
7. **Payment plan** — for off-plan (e.g. 50/50, 80/20, handover date)
8. **Master plan** — plan image + hotspots
